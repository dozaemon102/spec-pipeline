# コードレビュー チェックリスト

実装コードを詳細設計・要件定義と照合する。**AC 最終判定とトレーサビリティの完結は Phase 6（acceptance）が担当する。**

**レビュー対象**

- `projects/<project>/src/**/*`（割当 worker の担当パスのみ）
- （参照）`docs/features/<feature>/detailed-design/*.md`
- （参照）`docs/features/<feature>/requirements.md`
- （参照）`docs/architecture/tech-stack.md`

---

## Critical（次段階に進めない）

| ID | 確認項目 | 判定基準 |
|----|----------|----------|
| C-01 | API の未実装 | 詳細設計のエンドポイントが実装されていない |
| C-02 | API の仕様乖離 | メソッド・パス・リクエスト / レスポンスが設計と異なる |
| C-03 | DB の未実装 / 乖離 | テーブル・カラムが設計と一致しない |
| C-04 | 認可の未実装 | 必要ロールのチェックがないエンドポイントがある |
| C-05 | 認可の誤り | 要件の権限マトリクスまたは詳細設計のロールと実装が矛盾 |
| C-06 | エラーハンドリング | 設計のエラーコード（特に 403）が実装されていない |
| C-07 | スコープ逸脱 | 詳細設計にない機能・エンドポイントが存在する |
| C-08 | 配置違反 | コードが担当外パスに書かれている |
| C-09 | 必須テストの欠如 | 必須 FR に対応するテストが存在しない |

## Suggestion（人間判断で次へ進める）

| ID | 確認項目 | 判定基準 |
|----|----------|----------|
| S-01 | 画面の未実装 | 詳細設計 §4 に画面があるが UI が未実装 |
| S-02 | 画面の乖離 | 画面項目・バリデーションが設計と異なる |
| S-03 | worker 規約 | 割当 worker のコーディング規約からの逸脱 |
| S-04 | プロジェクト rules | `.agents/rules/` の規約違反 |
| S-05 | 命名 | 識別子が英語でない、または設計の命名と不一致 |
| S-06 | テストの薄さ | AC 紐づけがない、認可の片側のみテスト |
| S-07 | 未決事項 | 詳細設計の未決事項が実装で放置されている |

---

## レイヤ別の確認観点

### backend（`src/backend/`）

- 全 API エンドポイントと認可
- DB マイグレーション / モデル
- エラーレスポンス形式

### frontend（`src/frontend/`）

- 画面と API 呼び出しの対応
- フォームバリデーション
- 認可に基づく UI 制御（ボタン非表示等）

### infra（`src/infra/`）

- 詳細設計 / tech-stack に基づく設定
- 環境変数・シークレットの扱い（ハードコードなし）

---

## 判定ルール

- Critical が1件でもあれば **不合格**
- Suggestion のみなら **条件付き合格**
- 指摘なしなら **合格**

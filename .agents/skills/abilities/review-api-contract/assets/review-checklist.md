# API 契約レビュー チェックリスト

## Critical

| ID | 確認項目 |
|----|----------|
| C-01 | 詳細設計のエンドポイントが openapi.yaml にない |
| C-02 | openapi.yaml に詳細設計にないエンドポイントがある |
| C-03 | メソッド・パスが詳細設計と異なる |
| C-04 | 認可が必要な API に security 定義がない |

## Suggestion

| ID | 確認項目 |
|----|----------|
| S-01 | リクエスト/レスポンスの型が詳細設計と微妙に異なる |
| S-02 | types.ts が frontend 割当時にない |
| S-03 | x-usecase など詳細設計との紐づけがない |

## 判定ルール

- Critical が1件でもあれば **不合格**
- Suggestion のみなら **条件付き合格**
- 指摘なしなら **合格**

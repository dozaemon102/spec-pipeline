# セキュリティレビュー チェックリスト

## Critical

| ID | 確認項目 | 判定基準 |
|----|----------|----------|
| C-01 | シークレットの露出 | `.env`、API キー、パスワードのハードコード |
| C-02 | 認可バイパス | 権限チェックを迂回できる API・画面 |
| C-03 | 認証の欠如 | 保護すべきリソースが未認証でアクセス可能 |
| C-04 | 機密データの漏洩 | ログ・エラーレスポンスに PII / トークンが含まれる |
| C-05 | インジェクション | SQL / コマンドインジェクションの疑いがある実装 |

## Suggestion

| ID | 確認項目 | 判定基準 |
|----|----------|----------|
| S-01 | 入力検証 | 境界でのバリデーション不足 |
| S-02 | レート制限 | 公開 API に制限がない |
| S-03 | 依存の既知脆弱性 | 明らかに古い・危険な依存（確認可能な範囲） |
| S-04 | HTTPS / セキュア設定 | 本番想定の設定が infra にない |

## 判定ルール

- Critical が1件でもあれば **不合格**
- Suggestion のみなら **条件付き合格**
- 指摘なしなら **合格**

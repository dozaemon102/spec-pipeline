# 基本設計書レビュー チェックリスト

対象ファイルをすべて読み込み、各項目を確認する。

**レビュー対象**

- `docs/features/<feature>/basic-design.md`
- `docs/architecture/tech-stack.md`
- `docs/architecture/system-context.md`
- （参照）`docs/features/<feature>/requirements.md`

## Critical（次段階に進めない）

| ID | 確認項目 | 判定基準 |
|----|----------|----------|
| C-01 | 要件との対応 | 必須 FR のいずれかが設計要素に対応していない |
| C-02 | 詳細設計への混入 | API エンドポイント・DB カラム型・SQL が含まれている |
| C-03 | architecture の欠落 | `tech-stack.md` または `system-context.md` が存在しない |
| C-04 | 技術選定の根拠 | 技術スタックに選定理由がない |
| C-05 | 文書間の矛盾 | `basic-design.md` と `architecture/` の内容が矛盾している |
| C-06 | システム境界 | 本システムの範囲が明確でない |
| C-07 | 認証・認可 | 要件にロール・権限があるのに認可設計がない（不要時の「該当なし」明記もない） |
| C-08 | 権限の不整合 | 要件の権限マトリクスと認可設計が矛盾している |

## Suggestion（人間判断で次へ進める）

| ID | 確認項目 | 判定基準 |
|----|----------|----------|
| S-01 | トレーサビリティ | FR → 設計要素の対応表が未記載または不完全 |
| S-02 | NFR 対応 | 非機能要件への対応方針が薄い |
| S-03 | データ概要 | 主要エンティティが未定義 |
| S-04 | 外部連携 | 要件にある連携が設計に反映されていない |
| S-05 | 未決事項 | 要件の OPN が処理も引き継ぎもされていない |
| S-06 | 認可の具体性 | 認可方式はあるが、チェック実施層が未定 |
| S-07 | 変更履歴 | 差し戻し後なのに変更履歴が更新されていない |

## 判定ルール

- Critical が1件でもあれば **不合格**
- Suggestion のみなら **条件付き合格**
- 問題なしなら **合格**

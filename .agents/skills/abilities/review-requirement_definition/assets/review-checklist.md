# 要件定義書レビュー チェックリスト

各項目を確認し、問題があれば指摘を起票する。

## Critical（次段階に進めない）

| ID | 確認項目 | 判定基準 |
|----|----------|----------|
| C-01 | スコープ | 「含む」「含まない」の両方が記載されている |
| C-02 | 機能要件の存在 | 必須の機能要件が1件以上ある |
| C-03 | 設計への混入 | 機能要件に実装・API・DB の具体が含まれていない |
| C-04 | 受入条件 | 必須機能要件に対応する受入条件がある |
| C-05 | 受入条件の検証可能性 | 曖昧な表現のみ（「正しく動作する」等）で終わっていない |
| C-06 | 矛盾 | スコープ・機能要件・受入条件の間に矛盾がある |
| C-07 | 概要と要件の乖離 | 概要で述べた機能が要件に落ちていない |
| C-08 | ロール・権限 | ロール・権限の記載がない（不要時の「該当なし」明記もない） |

## Suggestion（人間判断で次へ進める）

| ID | 確認項目 | 判定基準 |
|----|----------|----------|
| S-01 | 非機能要件 | 性能・セキュリティ・可用性が未記載（該当なしの明記も可） |
| S-02 | 未決事項 | 曖昧な点が未決事項に列挙されていない |
| S-03 | 用語定義 | 専門用語が未定義のまま使われている |
| S-04 | 優先度 | 機能要件に優先度が付いていない |
| S-05 | ID の一貫性 | FR / NFR / AC / OPN の ID が重複・欠番 |
| S-06 | 変更履歴 | 差し戻し後なのに変更履歴が更新されていない |
| S-07 | ステークホルダー | ユーザー像・ステークホルダーが薄い |
| S-08 | 権限マトリクス | ロールは定義されているが、操作との対応（マトリクス）が未記載 |

## 判定ルール

- Critical が1件でもあれば **不合格**
- Suggestion のみなら **条件付き合格**（人間に提示）
- 問題なしなら **合格**
